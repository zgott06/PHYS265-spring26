Part 1: The Gravitational Potential of the Earth¶

The gravitational potential at a distance $r$ from a mass $M$ is given by:

$$\Phi(r) = -\frac{GM}{r}$$

where G is the gravitational constant.

For in that approximation, the the potential contains a singularity. In our function, use all modern best practices for coding. def grav_potent(M : float, x_M : float, y_M : float, x : float, y : float) -> float:
    """A function which calculates the gravitational potential at a distance r from a mass M

    Args:
        M   (float): mass of body M
        x_M (float): cartesian x coordinate of the mass M
        y_M (float): cartesian y coordinate of the mass M
        x   (float): cartesian x coordinate of evaluation point
        y   (float): cartesian y coordinate of evaluation point

    Returns:
        float: gravitational potential
    """

    # pre-conditions 
    if not isinstance(M, (int, float)):
        raise TypeError(f"Mass {M} should be a real number")
    elif not isinstance(x_M, (int, float)):
        raise TypeError(f"Mass x coordinate {x_M} should be a real number")
    elif not isinstance(y_M, (int, float)):
        raise TypeError(f"Mass y coordinate {y_M} should be a real number")
    elif not isinstance(x, (int, float, np.ndarray)):
        raise TypeError(f"x coordinate {x} should be a real number or numpy array")
    elif not isinstance(y, (int, float, np.ndarray)):
        raise TypeError(f"y coordinate {y} should be a real number or numpy array")
    elif M <= 0:
        raise ValueError(f"Mass {M} should be positive")

    # code body
    else:
        r = np.hypot(x_M - x, y_M - y)
        r = np.where(np.isclose(r, 0), np.nan, r) # resolve singularity
        result = -G * M / r

    # post-condition
    assert isinstance(result, (int, float, np.ndarray)), "The computed gravitational potential is not a real number or numpy array"
    
    return result

Part 3: The Gravitational Force Field of the Earth-Moon System¶

The gravitational force $\vec{F}$ that a mass $M_1$ exerts on a mass $m_2$ is:

$$\vec{F}_{21} = -G \frac{M_1 m_2}{|\vec{r}_{21}|^2} \hat{r}_{21}$$

where $\vec{r}_{21}$ is the displacement vector from $M_1$ to $m_2$.

  1. Have the function return the two gravitational force components, $F_x$ and $F_y$, experienced by the mass $m_2$ from the force exerted by $M_1$. Note that the gravitational force contains a singularity, and so if you made the approximation that the Earth is a finite point, you will need to avoid it. Like for the 1D potential, make sure it follows modern best-coding practices.

def grav_force(M1 : float, m2 : float, x1 : float, y1 : float, x2 : float, y2 : float) -> tuple:
    """A function which calculates the two gravitational force components experienced by a mass from the force exerted by another mass

    Args:
        M1 (float): mass of body M1
        m2 (float): mass of secondary body m2
        x1 (float): cartesian x coordinate of the mass M1
        y2 (float): cartesian y coordinate of the mass M1
        x2 (float): cartesian x coordinate of the mass m2
        y2 (float): cartesian y coordinate of the mass m2

    Returns:
        tuple: gravitational force components Fx and Fy
    """

    # pre-conditions
    if not isinstance(M1, (int, float)):
        raise TypeError(f"Mass M1 {M1} should be a real number")
    elif not isinstance(m2, (int, float)):
        raise TypeError(f"Secondary mass m2 {m2} should be a real number")
    elif not isinstance(x1, (int, float, np.ndarray)):
        raise TypeError(f"Mass M1 x coordinate {x1} should be a real number or numpy array")
    elif not isinstance(y1, (int, float, np.ndarray)):
        raise TypeError(f"Mass M1 y coordinate {y1} should be a real number or numpy array")
    elif not isinstance(x2, (int, float, np.ndarray)):
        raise TypeError(f"Secondary mass m2 x coordinate {x2} should be a real number or numpy array")
    elif not isinstance(y2, (int, float, np.ndarray)):
        raise TypeError(f"Secondary mass m2 y coordinate {y2} should be a real number or numpy array")
    elif M1 <= 0:
        raise ValueError(f"Mass M1 {M1} should be positive")
    elif m2 <= 0:
        raise ValueError(f"Mass M1 {m2} should be positive")

    # code body
    else:
        # compute distance
        dx = x2 - x1
        dy = y2 - y1
        
        r = np.hypot(dx, dy)
        r = np.where(np.isclose(r, 0), np.nan, r) # resolve singularity

        # resolve component directions
        r_hat_x = dx / r
        r_hat_y = dy / r

        # compute magnitude and Fx and Fy components
        F_mag = -G * M1 * m2 / (r**2)
        Fx = F_mag * r_hat_x
        Fy = F_mag * r_hat_y
        
        result = Fx, Fy

    # post-condition
    assert isinstance(result[0], (int, float, np.ndarray)), "The computed gravitational force component Fx is not a real number or numpy array"
    assert isinstance(result[1], (int, float, np.ndarray)), "The computed gravitational force component Fy is not a real number or numpy array"
    
    return result

Your Derivation Goes in the markdown cell below:

Given the magnitude of the gravitational force produced by a mass $M$ on secondary mass $m$ at distance $r$ is $$F=\frac{G M m}{r^2}$$

Let

  • $M_E$ = mass of the Earth
  • $M_M$ = mass of the Moon
  • $d_{EM}$ = distance from the Earth to the Moon
  • $r$ = distance from the Earth to the balance point
  • $d -r$ = distance from the Moon to the balance point

Now, set the force magnitudes produced by the Earth and the Moon (to the balance point) to be equal to one another $$\frac{G M_E m}{r^2}=\frac{G M_M m}{(d_{EM} - r)^2}$$

Cancel out $G$ and $m$ $$\frac{M_E}{r^2}=\frac{M_M}{(d - r)^2}$$

Take the square root of both sides $$\frac{\sqrt{M_E}}{r}=\frac{\sqrt{M_M}}{d_{EM} - r}$$

Cross-multiply $$\sqrt{M_E}(d_{EM} - r)=\sqrt{M_M} \, r$$

Expand terms $$\sqrt{M_E}d_{EM} - \sqrt{M_E}r=\sqrt{M_M}r$$

Group terms in $r$ $$\sqrt{M_E}d_{EM}=r(\sqrt{M_M} + \sqrt{M_E})$$

Solve for $r$ $$r=\frac{\sqrt{M_E}}{\sqrt{M_E} + \sqrt{M_M}} \, d_{EM}$$

Part 4: Altitude of the Saturn V Rocket¶

Rockets function by conservation of momentum. The ejection of fuel backwards propels the rocket forwards. The change in a rocket's velocity $\Delta v$ as a function of time $t$ (under a set of simplifying assumptions) is given by the Tsiolkovsky rocket equation:

$$\Delta v(t) = v_e \ln \left(\frac{m_0}{m(t)}\right) - gt$$

where $m_0$ is the initial "wet" mass (fuel + rocket parts + payload), $m(t) = m_0 - \dot{m}t$ is the mass at time $t$, $\dot{m}$ is the fuel burn rate (which we will assume to be constant), $v_e$ is the fuel exhaust velocity, and $g$ is the gravitational acceleration.

You can find the altitude $h$ of the rocket at "burnout" (when all the fuel is used up) by integrating the velocity over time:

$$h = \int_{0}^{T} \Delta v(t) \, dt$$

where $T$ is the total burn time of the rocket, which by conservation of momentum arguments is:

$$T = \frac{m_0 - m_f}{\dot{m}}$$

where $m_f$ is the final "dry mass" of the rocket, once all the fuel is burned.

def v(t : float, m_0 : float, m_f : float, m_rate : float, v_ex : float, g : float) -> float:
    """A function which calculates the change in a rocket's velocity as a function of time (Tsiolkovsky rocket equation)

    Args:
        t      (float): time 
        m_0    (float): wet mass
        m_f    (float): dry mass
        m_rate (float): burn rate
        v_ex   (float): exhaust speed
        g      (float): gravitational constant

    Returns:
        float: change in velocity
    """

    # pre-conditions
    if not isinstance(t, (int, float)):
        raise TypeError(f"Time {t} should be a real number")
    elif not isinstance(m_0, (int, float)):
        raise TypeError(f"Wet mass {m_0} should be a real number")
    elif not isinstance(m_f, (int, float)):
        raise TypeError(f"Dry mass {m_f} should be a real number")
    elif not isinstance(m_rate, (int, float)):
        raise TypeError(f"Burn rate {m_rate} should be a real number")
    elif not isinstance(v_ex, (int, float)):
        raise TypeError(f"Exhaust speed {v_ex} should be a real number")
    elif not isinstance(g, (int, float)):
        raise TypeError(f"Gravitational constant {g} should be a real number")
    elif t <= 0:
        raise ValueError(f"Time {t} should be positive")
    elif m_0 <= 0:
        raise ValueError(f"Wet mass {m_0} should be positive")
    elif m_f <= 0:
        raise ValueError(f"Dry mass {m_f} should be positive")
    elif m_rate <= 0:
        raise ValueError(f"Burn rate {m_rate} should be positive")
    elif v_ex <= 0:
        raise ValueError(f"Exhaust speed {v_ex} should be positive")
    elif g <= 0:
        raise ValueError(f"Gravitational constant {g} should be positive")

    # code body
    else:
        # end velocity contribution after burnout
        T = (m_0 - m_f) / m_rate
        if t > T: return 0

        # mass m as function of time t
        m = lambda t: m_0 - m_rate*t

        # Tsiolkovsky rocket equation
        result = v_ex * np.log(m_0 / m(t)) - g*t

    # post-condition
    assert isinstance(result, (int, float)), "The computed change in velocity is not a real number"
    
    return result

The burnout time is mainly dependent on the equation $T = \frac{m_0 - m_f}{\dot{m}}$ since this equation is directly based on fuel mass and burn rate, it is less sensitive to external effects like drag.

As for the altitude, the calculated burnout altitude was about 74.1 km, while the measured altitude was about 70 km. The model overestimates altitude by about 4 km. This is significant. The error in calculation could be due to a variety of factors. The model neglects atmospheric drag. The drag force opposes motion and therefore would reduce velocity and altitude. It can be noted that the effect of atmospheric drag is more significant in the lower atmosphere. Additionally, the model assumes constant gravitational acceleration, while in reality, gravitational acceleration decreases with altitude. This is a small effect, but it nonetheless introduces minor error. Another source of error in the model is that it assumes a constant exhaust velocity. In reality, the exhaust velocity produced by engines varies with pressure as the fuel is used up. The model also doesn't account for structural losses or inefficiens like steering correction. Put together, by not including these parameters in the calculations, the model is an overestimate to reality.
